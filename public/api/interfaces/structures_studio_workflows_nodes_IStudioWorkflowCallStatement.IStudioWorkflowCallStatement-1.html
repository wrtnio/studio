Interface IStudioWorkflowCallStatement

워크플로우상 커넥터 함수 호출문.

IStudioWorkflowCallStatement 는 IStudioWorkflow 워크플로우 프로그램 상 IStudioWorkflowConnector 커넥터 함수를 호출하는 표현식 문을 형상화한 노드 엔티티이다.

IStudioWorkflowCallStatement 를 구성할 때는 connector 로 호출하고자 하는 대상 커넥터를 지정하고, arguments 에 커넥터 호출시 할당할 파라미터 리스트를 AST 표현식으로 넣어주면 된다.

참고로 본 워크플로우 시스템은 커넥터 함수를 호출할 때마다, 그 리턴값을 모종의 지역 변수에 담아두고 있다. 그리고 이 모종의 지역 변수는 IAstWorkflowCallStatementResultIdentifier 로 식별된다. 다만, operand 을 명시적으로 지정한 경우, IStudioWorkflowVariable 에 저장되어 IAstWorkflowVariableIdentifier 로 식별해야 한다.

이외에 title 과 descrition 필드에 관해서는 슈퍼타입 IStudioWorkflowNodeDescriptable 를 읽어보도록 하자.

const result = connector(...arguments);

nexts

nexts: IStudioWorkflowNodeNext[]

다음 노드문의 리스트 정보.

현행 노드문의 실행이 완료된 이후, 그 다음 진행할 노드문들의 IStudioWorkflowStatement.id 와 그에 대한 설명을 적은 IStudioWorkflowNodeNext 구조체를 지칭하는 리스트.

참고로 지칭 가능한 노드는 현행 노드와 동일한 레벨의 블록에 속해야 한다. 만일 현행 노드가 IStudioWorkflowForEachStatement for loop 나 IStudioWorkflowBranchStatement if statement 등의 블록에 귀속된 노드를 지칭하거나, for loop 나 if statement 등의 블록에 귀속된 노드가 자신이 속한 블록 밖의 노드를 지칭한다면, 이는 워크플로우 컴파일러에 의하여 validation error 가 발생한다.

만약 이 값이 설정되지 않았다면, 커넥터 호출 결과는 오직 IAstWorkflowCallStatementResultIdentifier 로 식별한다.

Optionalparameter_types

parameter_types?: IStudioWorkflowConnectorParameter[]

파라미터 타입을 명시적으로 변형할 수 있다.

본래의 커넥터 함수의 파라미터와는 다른 타입을 명시적으로 지정하는 경우.

예를 들어 원래의 커넥터 함수는 any 타입의 variadic 파라미터를 수신하는데, 모종의 이유로 해당 커넥터의 파라미터 타입을 특정할 수 있게 되어 동적으로 해당 파라미터들의 타입 정보를 명시하고 싶은 경우.

예를 들어 notion DB 에 관련된 커넥터 등이 그러하다.
