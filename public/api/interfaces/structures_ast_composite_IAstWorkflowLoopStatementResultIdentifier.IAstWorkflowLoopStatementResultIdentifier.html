Interface IAstWorkflowLoopStatementResultIdentifier

워크플로우 Loop 문의 리턴값에 대한 식별자 표현식.

IStudioWorkflow 워크플로우 문서에서 IStudioWorkflowForEachStatement 이나 IStudioWorkflowWhileStatement 등의 반복문이 사용된 경우, 해당 반복문 블록 내에서 IStudioWorkflowCallStatement 커넥터 함수 호출 등으로 인하여 생성된 IAstWorkflowCallStatementResultIdentifier 모든 지역 변수들은, 반복문 블록 시작 전에 선언된 배열에 삽입된다.

단, 반복문 내에서 IStudioWorkflowBreakStatement break 나 IStudioWorkflowContinueStatement continue 문 등이 쓰인 경우라면, 지역 변수들은 배열에 삽입되지 않는다.

IAstWorkflowLoopStatementResultIdentifier 는 바로 이러한 반복문에서 생성된 지역 변수들을 담아둔 배열을 식별하는 표현식이다. 즉 IAstWorkflowLoopStatementResultIdentifier 는 IAstIdentifier 의 일종이되, 반복문 결과물 배열만을 위한 전용 식별자 표현식인 것.

물론 반복문이 이중 삼중으로 쓰인 경우에도, IAstWorkflowLoopStatementResultIdentifier 가 지칭하는 배열 변수 또한 그보다 상위 루프문의 배열에 삽입된다.

만약 위 내용이 잘 이해되지 않는다면, 아래 예제 코드를 살펴보자. 아래 예제 코드의 pointers 변수가 바로 본 IAstWorkflowLoopStatementResultIdentifier 가 지칭하는 대상인 것.

interface Pointer {
  x: number;
  y: number;
  z: number;
}
const pointers: Pointer[] = [];

for (let i: number = 0; i < 10; ++i) {
  // For loop 내에서 3 개의 커넥터 함수를 호출
  const x: number = computeX(i);
  const y: number = computeY(i);
  const z: number = computeZ(i);

  // continue 나 break 노드 존재시 해당 없음
  if (x === y && y === z) continue;
  else if (x < 0) break;

  // For loop block 이 끝나면서,
  // 생성된 지역변수들을 모두 배열에 담는다
  pointers.push({ x, y, z });
}

// 그리고 이를 For loop 바깥에서 활용 가능
const emended: IPointer[] = emendPointers(pointers);
